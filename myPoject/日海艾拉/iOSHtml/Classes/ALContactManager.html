Overview

Creates and manages the contacts for the session.

The contact manager is supported by a memory cache, when you first create or fetch a contact the data will be available inmediatly for subsequent calls. The cache can be manually updated if you call the refreshContactsWithSuccess:failure: method on this class, the method will update all the objects in the cache

To enumerate the contacts you must first call refreshContactsWithSuccess:failure: to update the cache, after that hasContacts will return YES, then you can use the numberOfContacts property to find out how many contacts exist and contactAtIndex:success:failure: to retrieve them.

The class also implements subscritped access, so contactManager[0] will return the first contact, but also calling contactManager[contactKey] is possible.
